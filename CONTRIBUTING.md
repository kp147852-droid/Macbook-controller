# Contributing

Thanks for contributing.

## Development setup
1. Clone repo
2. Set up relay (`relay/README instructions in root README`)
3. Open `apps/MacbookController.xcodeproj` for native clients

## Pull request process
1. Create a feature branch
2. Keep changes focused and documented
3. Update README/docs when behavior changes
4. Ensure CI passes
5. Open PR with clear summary and testing notes

## Coding expectations
- Security-sensitive changes require explicit rationale
- Avoid breaking E2E handshake compatibility without migration notes
- Keep logs free of secrets and sensitive payload content
